# Release 1.4.0

**Release date:** 2026-03-04

This minor release adds deployment hardening documentation, completes the web hardening plan, and archives completed plan documents. Application behaviour and APIs are unchanged; all tests pass with no errors or warnings.

## Changes since 1.3.1

### Documentation and structure

- **Completed plans archive:** All verified-and-complete plan documents (Minor data detection, Aggregated identification, Sensitive categories ML/DL, Rate limiting, Web hardening) were moved to **`docs/completed/`**. **`docs/PLANS_TODO.md`** remains in **`docs/`** as the single source of truth for project plan status; it now links to the archived plans in `docs/completed/`. All references across USAGE, sensitivity-detection, and minor-detection docs (EN and PT-BR) were updated to the new paths.

### Web hardening and security (docs)

- **Deploy hardening:** **`docs/deploy/DEPLOY.md`** gained an optional **“Security and hardening”** section: Docker (non-root, resource limits, healthchecks) and Kubernetes examples (securityContext, NetworkPolicy, PodDisruptionBudget). Optional example files added: **`deploy/kubernetes/network-policy.example.yaml`** and **`deploy/kubernetes/pdb.example.yaml`**.
- **SECURITY.md:** New section **“Deployment hardening and reverse proxy”** referencing CSP/headers in `api/routes.py`, the new DEPLOY.md hardening content, and recommendation to use a reverse proxy with TLS and WAF when exposed.
- **USAGE (EN/PT-BR):** New subsection **“API and security (CSP, headers)”** describing CSP defaults, Chart.js CDN, and how to use a stricter CSP via reverse proxy.
- **Man pages:** **lgpd_crawler.1** has a new **SECURITY** section; **lgpd_crawler.5** documents security headers in the `api` config description. Both point to SECURITY.md and docs/deploy/DEPLOY.md.
- **Help page:** New **“Security headers and hardening”** card with links to SECURITY.md and docs/deploy/DEPLOY.md.

### Version and Docker

- **Version:** Bumped to **1.4.0** (minor) in `pyproject.toml`, `core/about.py` fallback, `docs/lgpd_crawler.1`, `docs/lgpd_crawler.5`, and `docs/deploy/DEPLOY.md` examples.
- **Docker:** Image `fabioleitao/python3-lgpd-crawler:1.4.0` and `fabioleitao/python3-lgpd-crawler:latest` are built and pushed to Docker Hub.

## How to get 1.4.0

- **From source:** `uv sync` or `pip install -e .` after pulling the tag.
- **Docker:** `docker pull fabioleitao/python3-lgpd-crawler:1.4.0` or `docker pull fabioleitao/python3-lgpd-crawler:latest`.

## Full test suite

Run `uv run pytest tests/ -v -W error`; all tests must pass with no errors or warnings.
